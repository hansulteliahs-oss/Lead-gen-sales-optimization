  <action>
    **1. Create `app/[lccSlug]/layout.tsx` (Server Component):**

    Async Server Component that fetches the LCC's `name` and `slug` using `createAdminClient()`. If no LCC found for the slug, call `notFound()`. Renders `<LccWebNav>` above `{children}` wrapped in a `<main>`.

    Follow the exact pattern from the research (Pattern 1):
    ```typescript
    import { notFound } from 'next/navigation'
    import { createAdminClient } from '@/utils/supabase/admin'
    import LccWebNav from '@/components/LccWebNav'
    ```

    Props interface: `{ params: { lccSlug: string }, children: React.ReactNode }`

    The layout wraps ALL routes under `[lccSlug]/` including the landing page and thank-you page. Per CONTEXT.md, this is explicitly acceptable.

    **2. Create `components/LccWebNav.tsx` (Client Component):**

    Implement the full sticky nav following all locked decisions from CONTEXT.md. Use the research Pattern 2 as the implementation template. Key requirements:

    - `'use client'` directive at top
    - Props: `{ lccName: string, lccSlug: string }`
    - `useState` for `menuOpen` toggle
    - `usePathname()` for active link detection and CTA routing
    - Nav links: About, Au Pairs, FAQ, Testimonials (in that order)
    - LCC name is a `<Link>` to `/${lccSlug}` (home link, no separate "Home" item)
    - CTA "Get Started" button: use `<a>` tag (NOT `<Link>`). When on landing page (`pathname === /${lccSlug}` or `pathname === /${lccSlug}/`), href is `#form`. On sub-pages, href is `/${lccSlug}/#form`.
    - Active state: current page link gets `text-brand-gold` with a bottom border or font-medium (Claude's discretion per CONTEXT.md)
    - Sticky: `sticky top-0 z-50` with white background
    - Bottom edge: border or shadow (Claude's discretion)
    - Desktop: links visible via `hidden md:flex`
    - Mobile: hamburger button via `md:hidden`, toggles dropdown panel
    - Dropdown: `max-h-0` to `max-h-64` transition with `duration-200` for ~200ms slide
    - Hamburger icon toggles to X when open
    - Tapping any nav link calls `setMenuOpen(false)` to auto-close menu
    - CTA at bottom of mobile menu

    **3. Add `id="form"` to landing page form wrapper:**

    In `app/[lccSlug]/page.tsx`, find the element that wraps `<LeadCaptureForm>` and add `id="form"` to it. If there's no wrapping element, wrap the form in a `<section id="form">`. This enables the CTA scroll-to behavior.

    **4. Add `scroll-smooth` to root layout:**

    In `app/layout.tsx`, add the class `scroll-smooth` to the `<html>` element. This enables CSS smooth scrolling for anchor links (`#form`). If there's already a `className` on `<html>`, append to it.
